# Contributing

感谢你为 ClipWheel 做贡献。为了提高协作效率，请遵循以下流程：

1. 提交 Issue 描述问题或需求（包含系统版本、复现步骤、预期行为）。
2. 新建分支后开发，保持单一主题提交。
3. 本地通过 `build.bat` 和 `scripts\\smoke-test.bat`。
4. 提交 PR，说明改动动机、功能变化、潜在影响与测试结果。

## 代码规范
- C 源码使用 UTF-8。
- 保持函数职责单一，避免过长函数。
- UI 文字统一中文。
- 对复杂逻辑添加简洁注释。

## 提交前检查
- 能构建成功。
- 无临时文件、日志、编译产物混入。
- 文档已同步更新（README/CHANGELOG）。
